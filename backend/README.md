# 🖥️ SkillMetric Backend — Node, Express, MongoDB & Gemini API

This is the API server for **SkillMetric**. It handles file uploads, parsing, user authentication, database persistence, and integrates with the Google Gemini API to analyze, grade, and rewrite resumes.

---

## 🛠️ Tech Stack & Justifications

*   **Node.js & Express**: Provides a lightweight, fast, event-driven architecture that is perfect for processing file buffers (PDF, DOCX) and handling asynchronous HTTP request cycles.
*   **MongoDB & Mongoose**: A NoSQL document database. A document-based store is highly suited for storing resume text data and nested AI analysis objects (which contain dynamic arrays of suggestions, scores, and markdown strings) without complex schema migrations or slow SQL join queries.
*   **Multer**: Node.js middleware for handling `multipart/form-data`, which is used for uploading resumes. It restricts files to a maximum size of 5MB and validates mime types to secure the server.
*   **PDF-parse**: A library that extracts raw, unformatted text from PDF files directly from the upload memory buffer.
*   **Mammoth**: A library designed to convert Word document `.docx` files into clean, raw, unformatted text, avoiding XML clutter and metadata extraction issues.
*   **JSON Web Tokens (JWT) & BcryptJS**: Standard stateless authentication. User passwords are salted and hashed with `bcryptjs` before DB write, and API authorization is checked on routes via a JWT bearer token header.
*   **Helmet & CORS**: Helmet sets HTTP security headers (e.g. X-Content-Type-Options, X-Frame-Options, CSP) to harden the server, while CORS lets us safely link with the Vite dev server.

## 🤖 Google Gemini AI Core Integration (`/utils/aiEngine.js`)

The AI engine orchestrates communication with Google Generative AI to perform resume analysis.

### Detailed Workflow:
1.  **Model Selection**: It targets `gemini-2.5-flash` by default. Alternative models can be configured via the `GEMINI_MODELS` environment variable (which accepts a comma-separated fallback list, e.g., `gemini-2.5-pro, gemini-2.5-flash`).
2.  **Prompt Construction**: It wraps the raw resume text and target role in a structured prompt specifying the scoring rubric and requesting specific formatting (like the Google X-Y-Z formula for bullet suggestions).
3.  **Strict JSON Output**: We configure the model with `responseMimeType: 'application/json'`. This forces the Gemini API to output syntactically correct JSON matching our exact layout, making server parsing bulletproof.
4.  **Resiliency & Retries**: Handles transient error codes (like `429` Rate Limit Exceeded or `500/502/503/504` server errors) by repeating the request up to 3 times per model, applying exponential backoff delays.
5.  **High-Fidelity Offline Mock Fallback**: If `GEMINI_API_KEY` is not present, it intercepts the call and runs a local keyword scanner. It categorizes the role, matches technologies, generates realistic improvement suggestions, and returns a fully formatted mock analysis. This guarantees full offline testability.

## 📂 Database Schema Models (`/models/`)

### 1. User Model (`User.js`)
Stores user profiles and handles hashing passwords before save.
```javascript
{
  name: { type: String, required: true },
  email: { type: String, required: true, unique: true },
  password: { type: String, required: true, select: false },
  createdAt: Date,
  updatedAt: Date
}
```

### 2. Resume Model (`Resume.js`)
Stores reference links to local files and the extracted text payload.
```javascript
{
  userId: { type: ObjectId, ref: 'User', required: true },
  fileName: { type: String, required: true },
  fileUrl: { type: String, required: true },
  rawText: { type: String, required: true },
  createdAt: Date,
  updatedAt: Date
}
```

### 3. Analysis Model (`Analysis.js`)
Stores grading metrics and optimization data generated by Gemini.
```javascript
{
  userId: { type: ObjectId, ref: 'User', required: true },
  resumeId: { type: ObjectId, ref: 'Resume', required: true },
  targetRole: { type: String, required: true },
  atsScore: { type: Number, required: true },
  breakdown: {
    formatting: Number,
    keywords: Number,
    impact: Number,
    structure: Number
  },
  skills: {
    matched: [String],
    missing: [String]
  },
  bulletPointSuggestions: [
    {
      original: String,
      suggestion: String,
      benefit: String
    }
  ],
  generalSuggestions: [String],
  improvedResumeText: { type: String, required: true },
  createdAt: Date,
  updatedAt: Date
}
```

## 🌐 REST API Endpoint Reference

All routes under `/api/resumes` require a header: `Authorization: Bearer <JWT_TOKEN>`.

### Authentication Routes (`/api/auth`)
*   `POST /register`: Registers a new user. Expects JSON: `{ "name", "email", "password" }`. Returns JWT token.
*   `POST /login`: Logins user. Expects JSON: `{ "email", "password" }`. Returns JWT token.
*   `GET /me`: Returns logged-in user profile metadata. (Protected)

### Resume & Analysis Routes (`/api/resumes`)
*   `POST /upload`: Expects `multipart/form-data` with key `resume` containing the PDF, DOCX, or TXT file. Returns Resume Metadata.
*   `POST /analyze/:id`: Starts the Gemini analysis pipeline. Expects JSON: `{ "targetRole": "..." }`. Returns the generated `Analysis` report.
*   `GET /history`: Returns a combined listing of user's uploaded resumes and their analyses.
*   `GET /analysis/:id`: Retrieves full details of a specific analysis record.
*   `DELETE /:id`: Deletes the resume record, deletes the local file from server disk, and clears all associated analyses.

## 🛠️ Step-by-Step Installation & Run Guide

### 1. Prerequisites
Ensure you have **Node.js** (v18+) and **MongoDB** installed and running on your system.

### 2. Install Dependencies
Navigate to the `/backend` folder and install packages:
```bash
npm install
```

### 3. Environment Variable Setup
Create a `.env` file in the `/backend` folder:
```env
PORT=5000
MONGO_URI=mongodb://127.0.0.1:27017/resume-grader
JWT_SECRET=yoursecretphrasehere
GEMINI_API_KEY=AIzaSy... (Get yours from Google AI Studio)
```

### 4. Running the Server

#### Development Mode:
Runs the server with auto-reloading if nodemon is configured, or standard node execution:
```bash
npm run dev
```

#### Production Mode:
Runs the server directly:
```bash
npm start
```

> [!NOTE]
> Static files are served from `/uploads` directory under the route `/uploads/:filename`. This folder will be created automatically by Multer upon the first file upload.
